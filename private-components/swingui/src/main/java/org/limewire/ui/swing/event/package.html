Overview

This package provides tools and a base class for using the EventBus library for GUI event messaging.

Implementation

The EventBus message subscription can be added to an object instance by adding a call to EventAnnotationProcessor.subscribe(this); within the constructor of an object. Within the subscription process, all public methods that have been decorated with an EventBus-specic annotations (org.bushe.swing.event.annotation.EventSubscriber, org.bushe.swing.event.annotation.EventTopicSubscriber, org.bushe.swing.event.annotation.EventTopicPatternSubscriber, org.limewire.ui.swing.event.RuntimeTopicEventSubscriber, or org.limewire.ui.swing.event.RuntimeTopicPatternEventSubscriber) will be connected to the EventBus (via a proxy object) and will afterward receive callbacks specific to the type of message subscription with which they are decorated.

There are a few classes in this package that are notable:

AbstractEDTEvent

This is the base class for events that need to be passed along on the EventDispatchThread. Events passed along for either topics, topic patterns, or just simple subscription should extend this class. The subclasses (by default) will publish to the Swing event service but the implementation of the publishing service can be easily changed (via a method defined here). Changing which event service is used can be useful for unit testing.

EventAnnotationProcessor

This class uses reflection to discover which methods in a class have one of the EventBus-related annotations. The class provides methods for event subscription and unsubscription.

RuntimeTopicEventSubscriber

This annotation allows classes to define the topics they would like to subscribe to at runtime by specifying a callback method in the annotation declaration that will be queried as the subscribing object is registered with the event bus.

RuntimeTopicPatternEventSubscriber

This annotation allows classes to do the same style of runtime subscription declaration that RuntimeTopicEventSubscriber supports, except that (regular expression) topic patterns provide the indication for which event messages should be sent to the subscribing annotated methods.
