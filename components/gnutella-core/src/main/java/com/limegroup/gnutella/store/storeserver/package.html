This is the main sub-package for store serevr integration. There is one instance of the store server (IStoreServer) in RouterService, and one can attach one handler for a particular command, and zero or more listeners for a certain command.

A IStoreServer.Handler will respond to a command with a String response. A IStoreServer.Listener will not respond to a command but will simply do something.

An example of adding a listener would be the following. Here we add a listener to receive events called Play and then call the method play(), assuming it's defined somewhere. 

RouterService.getStoreServer().registerListener("Play", new AbstractListener("play") {
  public void handle(Map<String, String> args) {
    MediaPlayerComponent.instance().play();
  }
});

Likewise, we could register a handler to report back the state of the current player. 

RouterService.getStoreServer().registerHandler("GetState", new AbstractListener("get.state") {
  public String handle(Map<String, String> args) {
    return MediaPlayerComponent.instance().getStateString();
  }
});

Note. That Play and play (as well, GetState and get.state) are not the same and don't need to. The capitlized ones refer to the command to which this handler or listener will react; the lower case ones are just names, which currently aren't used.
