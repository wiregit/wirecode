Overview

These tests are integration tests for testing both the connection of the com.limegroup.gnutella.ls.server package with the org.limewire.lws.server , but also the this package with the rest of the Lime Wire core and GUI modules.

Simple tests

There are a number of tests -- those that have the same name as the ones in the lwsserver component tests that are used to assure that the authentication scheme still holds when you incorporate the CORE, GUI, and other modules.

Handler and Listener tests

These tests test the following invariants. Here, handler means LWSManager.Handler and listener means LWSManager.Listener .

  • You can only register one handler per command String , because each handler has to respond to a command.
  • You can register zero or more listeners for a String command. They simply need to do something.
  • After registering a handler or listener, and then sending a message to the LWS Manager, you should be guaranteed to have received that message.
  • After unregistering a handler or listener, and then sending a message to the LWS Manager, you should be guaranted to have NOT received that message.
  • The arguments that your pass the LWS Manager should be exactly those that are read by the handler of listeners.
